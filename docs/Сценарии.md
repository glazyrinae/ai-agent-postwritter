# Сценарии

## Сгенерировать outline статьи

1. Клиент вызывает `POST /articles/outline` с заголовком `Authorization: Bearer <token>`.
2. LangChain собирает prompt с фиксированным форматом markdown.
3. `writer` генерирует outline через backend `vLLM`.
4. Сервис парсит markdown в структурированные секции.
5. Клиент получает title, markdown outline и список секций.

## Сгенерировать полную статью

1. Клиент вызывает `POST /articles/generate` с заголовком `Authorization: Bearer <token>`.
2. Сервис создает `article_run` в PostgreSQL и сохраняет `run_id`.
3. Сервис сначала вызывает внутренний этап outline generation.
4. Для каждой секции сервис через LangChain вызывает `writer` для генерации содержимого главы.
5. После каждой главы сервис через LangChain вызывает `summarizer` для короткого summary.
6. При генерации следующей главы сервис передает topic, outline и summary уже написанных разделов.
7. После всех секций сервис генерирует заключение и прогоняет статью через `editor`.
8. Финальный результат возвращается как единый Markdown-документ и как список отдельных секций.
9. Если клиент потерял соединение, результат можно позже получить через `GET /articles/runs/{run_id}` и `GET /articles/runs/{run_id}/result`.

## LangChain pipeline

1. Клиент вызывает `POST /pipeline` с заголовком `Authorization: Bearer <token>`.
2. Сервис строит LangChain pipeline из списка agent aliases.
3. Каждый шаг пайплайна вызывает соответствующий agent alias в `vLLM`.
4. Результат последнего шага возвращается как итоговый текст пайплайна.

## Healthcheck

1. Оркестратор вызывает `GET /health` без авторизации.
2. Сервис отвечает базовым статусом для liveness/readiness-проверки.
