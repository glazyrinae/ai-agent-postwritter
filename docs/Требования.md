# Требования

## Функциональные

- Сервис должен поднимать low-level API для вызова отдельных агентов и LangChain pipeline.
- Сервис должен предоставлять отдельные endpoints для генерации outline и полной статьи.
- Генерация длинной статьи должна работать поверх существующего `writer`-агента.
- Оркестрация article flow должна быть реализована через LangChain.
- Оркестрация pipeline-слоя должна быть реализована через LangChain.
- Если outline не передан, сервис должен генерировать его сам.
- Для каждой новой главы контекст должен строиться на базе summary предыдущих разделов, а не полного текста статьи.
- Все основные API endpoints, кроме healthcheck, должны быть защищены Bearer token авторизацией.

## Нефункциональные

- Проект должен запускаться через Docker Compose.
- Код runtime API должен жить в `src/`.
- Ошибки API должны возвращаться в стабильном JSON-формате.
- Конфигурация должна управляться через environment variables.
- Контейнеры должны запускаться от `UID/GID`, задаваемых через env, со значениями по умолчанию `1000:1000`.

## Ограничения модели

- Базовая модель может принимать длинный входной контекст, но генерация за один запрос ограничена `max_tokens`.
- Поэтому длинная статья должна собираться итеративно по разделам.
- Пустой или слишком короткий ответ модели должен считаться ошибкой генерации.
